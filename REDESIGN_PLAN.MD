# Momito Redesign ‚Äî Phone + Laptop Interview-Prep App

## Context

Momito (D:\Momito\Momito-clone) is an interview-prep + career app the user is betting everything on: they are studying to pass interviews abroad and emigrate. Today it is a pnpm monorepo ‚Äî Next.js 16 web (`apps/web`), NestJS 11 + Prisma/Postgres API (`apps/api`, 26 models, ~75 endpoints), shared types (`packages/shared`) ‚Äî plus an **abandoned** Python FastAPI `backend/` and a 2-screen Expo `mobile/` wired to that dead backend.

The data model is strong, but the product has critical gaps:
- **Unusable on phones**: primary nav is `hidden sm:flex` with no mobile menu (`apps/web/app/(authenticated)/layout.tsx:50`); no PWA manifest/SW/icons; broken vestigial dark mode + `font-family: Arial` override in `apps/web/app/globals.css`.
- **Weak learning core**: 38 conceptual seeded questions (0 DSA), plain-`<textarea>` answers, no markdown rendering, no code editor, no timers. Spaced repetition (`spaced_review`, `needsReview`) and AI feedback (`aiScore`/`aiFeedback`) exist as dead schema columns ‚Äî never implemented.
- **Sprawl**: 12 flat nav links, three overlapping task systems (StudyPlanItem / Task / PlanItem), reminders that never fire (no cron), naive keyword-matching readiness scoring.
- **Git risk**: only 1 commit exists; the entire Career OS/missions layer (~40 files, 3 migrations) is uncommitted.

**User decisions (fixed):** ‚ëÝ PWA ‚Äî one responsive Next.js app, archive Expo + Python backend. ‚ë° Free cloud hosting (Vercel + Render + Neon) so the phone works anywhere. ‚ë¢ AI grading via Claude API is a core feature (budget-capped, graceful fallback). ‚ë£ All four domains built deep, in order: DSA/coding ‚Üí system design + CS fundamentals ‚Üí behavioral story bank ‚Üí jobs/CV pipeline.

**Evidence-based prep practices this redesign encodes:** FSRS spaced repetition (active recall at optimal intervals), pattern-based DSA (interleaved, timed, with error journaling), structured system-design frameworks with rubrics, STAR story vault with rehearsal, a single "Today" queue (one decision: what do I do right now), consistency streaks, and rubric-based feedback (AI-graded, self-rating fallback).

Each phase ends with a git commit and green `pnpm lint && pnpm typecheck && pnpm test && pnpm build`.
