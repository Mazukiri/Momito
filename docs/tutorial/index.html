Momito

Momito is your personal, single-user interview-prep tool. It merges spaced-repetition practice, coding drills, system-design rehearsal, behavioral storytelling, career-goal tracking, and job-application management into one daily loop, so you always know exactly what to do next.

This guide walks through every screen and feature, with real screenshots taken directly from a running instance. Use the left-hand menu to jump to any section, or just read top to bottom for a full walkthrough of a typical week of use.

Core idea Almost everything in Momito eventually funnels into one screen: Today. Reviews that are due, tasks that are overdue, job deadlines, and career-readiness gaps all show up there, ranked by urgency, so you never have to remember to check five different tabs.

Quick Start

Momito runs as two local servers: an API (NestJS) on port 3001 and a web app (Next.js) on port 3000.

1. Start the servers

pnpm --filter @momito/api dev

pnpm --filter @momito/web dev

2. Open the app

Visit http://localhost:3000 in your browser.

3. Log in

Email: demo@momito.local
Password: MomitoDemo123!
Single-user by design Momito is built as a personal tool, not multi-tenant SaaS. Once one account exists, new registrations are refused by default — this stops a stranger from signing up if you ever deploy the app publicly. Set ALLOW_MULTI_USER_REGISTRATION=true on the API if you want to allow more accounts.

Today (Home Screen) Start here every day

Today is the screen you land on after logging in. It shows a single priority-ranked queue that merges everything that needs your attention right now:

  • Due spaced-repetition reviews — questions or Story Bank entries whose schedule says "review now," ranked highest (and higher still the longer they've been overdue).
  • Recommendations — active missions, overdue tasks, career-readiness gaps, job deadlines, and unread learning highlights, each with a plain-English reason ("3 tasks overdue") and a badge showing where it will take you.
  • Reminders — dated nudges tied to tasks. Overdue ones rank near the top; future ones sink to the bottom but stay visible.

The streak badge

The 🔥 badge in the top-right counts consecutive days with at least one submitted answer. It doesn't break at midnight — if you haven't answered anything yet today, it still shows yesterday's count, giving you the rest of the day to keep it alive.

How to use it

  1. Open the app — you land on Today automatically.
  2. Work the queue top to bottom; it's already sorted by urgency.
  3. For a due review card, tap "Review now" — an inline 1–5 star strip appears. Rate your recall and it reschedules automatically, no need to open a full session.
  4. For anything else, tap the card to jump straight to the relevant screen (a job, a mission, the reading inbox, etc).
  5. Dismiss reminders you no longer need with the Dismiss button.

Practice Sessions

The Practice tab is where actual studying happens: answering questions, writing code, and rehearsing system designs, one question at a time inside a timed session.

The Practice Hub

Landing screen for the tab. Shows any session you already have in progress (tap to resume) plus four quick-start cards, and a link to your full answer history.

Starting a new session

Pick a Session Type — each has a different question-selection strategy:

TypeWhat it pulls
Quick PracticeRandom questions across all topics
Topic PracticeAll from one topic you pick
Company PracticeQuestions tagged with one company
Mixed MockMixed types and difficulties, like a real loop
Role DrillFiltered by career role track + focus area (+ optional pattern like "graph" or "dp")
Weak Area ReviewTopics where your self-ratings are lowest
Daily Mixed SetA small, balanced daily set
Job PrepPractice tied to one target job
Spaced ReviewThe exact set of items currently due for FSRS review (not random) — the form shows a live due-count and preview titles; disabled if nothing is due

You also choose optional difficulty and the number of questions (1–100, default 5).

Running a session

One question per screen, with a progress bar, a per-question timer, and an Abandon button if you need to bail (answers you already gave are kept). The answer input changes shape by question type:

Below the answer box:

  • Self Rating (optional, 1–5) — how well you think you did on this particular answer.
  • Reflection — collapsed by default behind "+ Add reflection." Expanding it reveals toggleable miss-tags (misread the question, wrong approach, missed edge case, ran out of time, didn't know where to start, implementation bug, didn't know the concept, struggled to explain it, missed a trade-off) plus a free-text note.

Submit Answer saves the attempt and auto-advances. A ← Previous button lets you step back and revisit ("Update Answer" replaces the label once a question already has an answer). Once every question is answered, a numbered-pip row lets you jump back to review any of them, and Complete Session finalizes it.

Session summary After completing, you land on a summary page listing every question with its status, your answer (truncated), time spent, self-rating, and reflection tags — with a link back into each question's full detail page.

DSA Ladder

A dedicated progress tracker across ~20 named coding-interview patterns (two pointers, sliding window, graph traversal, dynamic programming, and so on). Each pattern card teaches, not just labels:

  • Solved/total count with a color-coded progress bar (gray → amber → green).
  • A "When to use" one-liner explaining when that pattern actually applies.
  • A Practice → link that jumps straight into a quick session pre-filtered to that one pattern.

Questions Library

A searchable, filterable list of every question in your bank (search by title; filter by type, difficulty, topic), with pagination.

Question detail

Opening a question shows the full prompt with type/difficulty/topic/company badges, a collapsible Reference Answer rendered as formatted markdown (including syntax-highlighted code and tables), a one-click Start Practice button that spins up a 1-question session just for it, and — for behavioral questions — a panel to link Story Bank entries as prepared answers.

You can also Edit or Delete a question, or add a brand-new one (title, prompt, type, difficulty, topic, companies, reference answer, notes, source URL).

Answer History

Every answer you've ever submitted, newest first, with a snippet of the text, your self-rating, and whether it came from a session.

Opening one shows the original question, your full answer, your self-rating stars, your reflection tags/note if any, an AI Feedback card (see below), and a link back to the session it came from.

Spaced Repetition (FSRS)

Momito schedules reviews using FSRS, the same algorithm family used by flashcard apps like Anki. Every question or Story Bank entry you rate gets a "review state" tracking memory stability/difficulty and a computed due date.

Rate 1 → Again· Rate 2 → Hard· Rate 3–4 → Good· Rate 5 → Easy

Rating something low reschedules it sooner; rating it high pushes the next review further out. This is a separate rating channel from the practice-session self-rating — only ratings given through Today's "Review now" or a Spaced Review session actually reschedule anything. Both Questions and Story Bank entries share the same due-review queue, so they show up together on Today.

Practical effect You never have to decide what to re-study. If it's due, it appears on Today. If you keep rating something Easy, it fades from view for longer and longer stretches; if you keep struggling, it comes back fast until it sticks.

AI Grading Optional

An optional feature that only appears if whoever runs the API has configured an ANTHROPIC_API_KEY. If it's not configured, the feature is completely invisible — nothing breaks, no error shows, the app just works entirely on self-rating.

When available, an "AI Feedback" card appears on an attempt's detail page:

  • A "Grade with AI" button, if not yet graded.
  • Once graded: an overall score out of 100, a per-criterion breakdown against the question's rubric (if it has one), a Strengths list, a Gaps list, and likely interviewer follow-up questions based on your specific answer.
  • A Regrade link to re-run grading (bypassing the cache) — useful after editing your answer.
Behind the scenes Grading is idempotent (repeat requests return the cached result unless you force a regrade) and subject to a per-user daily USD budget — if exceeded, further requests fail cleanly with a budget message rather than erroring. The model compares your answer against the question's prompt, type, rubric, and reference answer.

Career Tracks

Pick a target role track (e.g. "Big Tech SWE," "Quant SWE," "HPC/GPU Engineer") from the dropdown and click Activate. Every track renders as a card with a checklist-item count and an Active/Inactive badge.

Each active track then shows a live readiness panel: an overall percentage bar, a per-area breakdown (DSA, system design, behavioral, etc. — each with its own bar), computed from your actual practice/review/mission activity, plus a "Top gaps" list of specific next actions.

How to use it Pick your target role → Activate it → check back periodically to watch your readiness score climb (and your top-gaps list shrink) as you practice.

Job Tracker

A filterable card list (not a kanban board) for tracking real applications. Add Job captures company, role, role track, deadline, URL, and a pasted job description. A status pill-bar filters by pipeline stage with live counts.

Job detail — three key actions

ButtonWhat it does
Create/Open MissionSpins up (or jumps to) a Mission tied to this job, sharing its role track and context.
Generate PrepAuto-generates prep tasks from the job description, which then surface on Calendar/Today.
Score ProfileScores your Profile against this specific job's JD and role track, then opens the resulting score breakdown.

The detail page also has a manual Timeline (interviews, notes, offers with timestamps), a status selector, and a Reminders list scoped to this job.

Missions The execution engine

Missions tie together diagnosis, a weekly plan, evidence, and review into one execution loop for a target role or job. Create one directly, or spin it off from a Job (auto-linking the job application).

Diagnose→ Generate weekly plan→ Work the plan (Practice / Learning / Calendar)→ Weekly Check-in→ Review latest plan
  1. Diagnose analyzes your current state against the mission's target, producing a diagnosis summary and a per-competency status (ready / building / gap), each with an evidence count and progress bar.
  2. Generate weekly plan produces a dated, typed task list (practice, learning, etc.) each with estimated minutes and an "expected artifact" to produce.
  3. Work through the plan using the mission's own quick links into Practice (pre-filtered to its role track), Learning (pre-filtered to this mission), and Calendar.
  4. Log a Check-in each week — summary, wins, blockers, adjustments — building a running history.
  5. Review latest plan closes out the cycle so the next Diagnose reflects what you actually did.

Study Plan

A simple personal to-do list for study tasks (backed by the same Task model used on Calendar). Three tabs — To Do / In Progress / Done — with live counts. + Add Task creates one with a title, optional topic tag, notes, and target date (overdue dates highlight red, due-soon highlights amber).

Story Bank Behavioral prep

Stores STAR-formatted stories (Situation / Task / Action / Result + optional Metrics) for behavioral interviews.

When creating or editing a story, tag it via a competency chip picker (ownership, leadership, conflict, ambiguity, etc.) and a company chip picker, plus optional anticipated follow-up questions.

Rehearsing a story

Click Rehearse — a 1–5 rating row appears. Recall the story out loud, then rate how it went.

This feeds FSRS too Rehearsal submissions call the same spaced-repetition endpoint used for practice questions, so rehearsed stories get scheduled and resurface automatically on Today when they're due — exactly like a coding or behavioral question review.

Learning Ledger

A permanent record of everything you've read or learned, optionally scoped to a Mission. Two ways to add evidence:

  1. Readwise integration — paste a Readwise access token to connect, then Sync Now pulls in your highlights.
  2. Manual evidence — a direct form (title, role track, area, notes) for logging any learning event.

Reading Inbox

The triage view for freshly-synced Readwise highlights. Pick a role track + area, then either Review (turns it into ledger evidence) or Ignore it. Once triaged, it disappears from the inbox.

Profile & CV Scoring

Profile is your structured, editable CV data: contact info, skills, experience/projects/education. Stat cards up top show entry counts.

Profile Scores

Runs your Profile against a role template or a pasted job description, producing four progress bars: Skills Match, Project Quality, Experience Depth, Presentation. Clicking into a score shows a breakdown per category with specific gaps and top-level suggestions.

How to use it Upload your CV PDF once (auto-extracts into the structured fields above — review and Save), then go to Profile Scores → pick a role template or paste a specific JD → review the four-category breakdown to know exactly what to strengthen. A Job's own "Score Profile" button does this same scoring pre-targeted at that job.

Dashboard Read-only overview

An analytics view, separate from the action-oriented Today screen — nothing here is editable, it's purely for checking in on trends.

Includes stat cards (questions practiced, sessions completed, weak topics, topics covered), a Focus Mission card, Role Readiness bars, Next Actions, This Week's tasks, Topic Progress bars, Weak Areas, and Recent Sessions.

Calendar

All scheduled tasks and reminders in one place, filterable by Today / This Week / Overdue / All. Add Task lets you set type, priority, role track, area, and due date.

Settings

Manages the two shared taxonomies used to tag questions:

  • Topics manager — add/edit/delete topics (e.g. "System Design," "Dynamic Programming"). Deleting a topic still linked to questions fails safely rather than orphaning data.
  • Companies manager — add/edit/delete companies (name, region, notes) used to tag questions by employer.

Content Coverage

A read-only report comparing your question bank against target content goals: total questions, company/role-track counts against targets, per-domain progress bars, and breakdowns by type and difficulty.

Account, Auth & Navigation

Login and registration are plain email + password (6+ characters). A shared top header always shows your name, a 🔔 reminder bell, a ☀️/🌙 dark-mode toggle, and Sign out.

Installing & Offline Use

Momito is installable as a Progressive Web App. On a phone browser, use "Add to Home Screen" to install it like a native app.

A production-only service worker precaches the app shell. Navigation always tries the network first (your data must stay fresh) and only falls back to a friendly offline page if the network request fails entirely. Your actual study data is never cached — only the static app shell — so nothing appears out of date.

Known Issues

Dark mode toggle doesn't restyle the page Clicking the ☀️/🌙 toggle correctly flips internal state and saves it to localStorage (confirmed via the dark class being applied to <html>), but the page's colors don't actually change — background stays light even after reloading. This looks like a CSS cascade/specificity issue with the custom Tailwind v4 dark variant rather than a state bug. Worth a follow-up fix; not covered in this guide since it was discovered while writing it, not requested as something to repair.
