# Skill — UX Polish

Use this skill when improving frontend.

Product personality:

- serious
- calm
- focused
- productivity-oriented
- suitable for interview preparation

Every main page should have:

- title
- short explanation
- primary action
- loading state
- error state
- empty state
- useful navigation

Empty states should tell the user what to do next.

Bad empty state:

```txt
No data.
```

Good empty state:

```txt
No questions yet. Seed sample questions or create your first backend interview question.
```

Forms should have:

- labels
- validation feedback
- disabled submitting state
- success/error feedback
- confirmation for destructive actions

Avoid flashy UI. Make it feel like a tool a serious student would use daily.
