# Skill — Productization

Use this skill when moving Momito from MVP to complete product.

Productization means:

- real first-run experience
- no false README claims
- complete CRUD for core entities
- helpful empty states
- seed/demo data
- stable local setup
- quality gates
- deployment-aware config
- coherent UX

Do not chase novelty before core flows are complete.

Priority order:

1. Fix broken or misleading flows.
2. Complete missing UI for existing backend capabilities.
3. Add seed data.
4. Add setup scripts.
5. Harden config.
6. Improve tests.
7. Polish UX.
8. Document everything.

Signs of incomplete product:

- route listed in README but not implemented
- API exists but UI cannot use it
- new user sees only empty screens
- setup requires hidden knowledge
- no seed data
- no env examples
- no way to verify quality
